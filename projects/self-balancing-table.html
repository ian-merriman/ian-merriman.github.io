Self-Balancing Table Project

A controls-focused prototype designed to practice real-time stabilization using sensor feedback, actuator response, mechanical design, and iterative hardware testing.

Control Systems Sensor Feedback Actuator Control Prototyping System Integration

Project Overview

The self-balancing table project was created to explore how a physical system can use sensor feedback and actuator response to maintain a desired orientation. The project focused on stabilization, control logic, mechanical design, and the practical challenges of turning a controls concept into a working prototype.

This project helped connect control theory to hardware implementation. Instead of only studying feedback control mathematically, the table provided a hands-on way to observe how sensor readings, actuator commands, mechanical motion, and disturbances interact in real time.

Demonstration GIF showing the self-balancing table prototype responding to changes in orientation.

My Role

I worked on the concept development, system planning, hardware integration, and control-focused thinking behind the project. My role involved understanding how the system should sense tilt, determine the needed correction, and command the actuator response needed to move the table back toward level.

I also used the project to practice troubleshooting and iterative testing, since small changes in mechanical setup, sensor placement, or control response can affect the behavior of the full system.

What I Learned

This project helped me understand how control systems behave differently when implemented on real hardware. Sensor data can be noisy, actuators have physical limits, and mechanical design choices can strongly influence whether the system responds smoothly or becomes unstable.

I also learned that successful control projects require more than code. They require careful system integration, mechanical reliability, good sensor placement, realistic actuator commands, and repeated testing.

Future Improvements

Future improvements could include adding a more formal PID controller, logging sensor and actuator data, improving the mechanical linkage, reducing sensor noise, adding a better calibration routine, and creating a block diagram or simulation model to compare expected behavior against hardware test results.
